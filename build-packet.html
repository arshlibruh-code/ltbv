01

Final Calls

Post-research. Supersedes the old PTY plan entirely.

CaptureStop hooks in both CLIs deliver last_assistant_message as JSON on stdin. No PTY, no ANSI parsing, ever.
TTSBench gate: Pocket TTS, then Supertonic-3 int8, then Kokoro-82M via mlx-audio. First to pass <1s warm + <1GB RAM wins.
DaemonLocalhost HTTP on 7333. Owns cleanup, condense, synthesis, playback. Lazy model load, exits after 10 min idle.
HookDumb forwarder. POSTs raw event + text to the daemon and exits in tens of ms. No network, no logic, no uv resolution: invoked as .venv/bin/python by absolute path.

Architecture

The whole thing is a hundred-ish lines.

Agent CLICodex 0.142.5 or Claude Code. Untouched, unwrapped.
Stop hookFires on turn end. JSON on stdin carries last_assistant_message.
hook.pyDumb forwarder: POST {event, text, ts} to daemon, auto-start it if down, exit. Nothing else.
daemon.pyPort 7333. /speak, /stop, /health. Cleanup + DeepSeek condense + synthesis, all async. Lazy load, 10 min idle exit.
PlaybackOne active + one pending, newest wins, generation IDs gate stale requests. UserPromptSubmit hits /stop.

05

Invariants

Never break these. Everything else is the model's job.

Never block the agentHook exits within ~100ms: it only POSTs and dies. All cleanup, condensing, synthesis, and playback happen async in the daemon. Hook is invoked as .venv/bin/python by absolute path (warm uv run measured 40-90ms, too knife-edge).
Race-safeDaemon start is bind-winner: both CLIs may fire at once, first to bind :7333 wins, loser just POSTs. Generation counter on utterances: a stale /speak can never resurrect speech after a newer /stop. /stop kills the whole afplay process group.
Silence over noiseAny failure (daemon down, API error, malformed payload) means no sound and exit 0. Errors never leak into the agent session.
Never speak codeNo code, diffs, stack traces, or raw paths, under any condition.
Instant interruptNew user input kills current speech immediately.
Local, one asteriskEverything runs on-device except the single DeepSeek condense call.
Stay smallNo framework, no packaging, no config UI, no tests beyond the bench script. A handful of files. Personal tool, not a product.
SecretsDeepSeek token read at runtime from ~/.claude/settings.deepseek.json. Never stored or committed in this repo.

Codex Prompt

Paste this. It restates everything above as one instruction block.

final prompt
Build a local voice layer for my coding agents (Codex CLI 0.142.5 and Claude Code
2.1.201). Repo: this folder (let-there-be-voice). Python 3.12 via uv. macOS on
Apple M3, 16GB RAM. This packet is v3, already amended by your own review of
2026-07-05: condense lives in the daemon, hook is a dumb forwarder, bench order
is Pocket / Supertonic / Kokoro.

Do NOT build a PTY wrapper or parse terminal output. Both CLIs expose the final
response via hooks: the Stop event delivers JSON on stdin with a
last_assistant_message field (keys confirmed present in both binaries via
strings scan). That is the only input source.

PHASE 0 - VERIFY (quick sanity, before real code):
- Dump one real Stop payload from each CLI (temp hook appending stdin to
  /tmp/hookdump.json, one real turn each) to confirm last_assistant_message is
  POPULATED at runtime, not just present as a key. Fallback if empty: parse
  transcript_path for the last assistant entry.
- Confirm the DeepSeek token is readable at runtime from
  ~/.claude/settings.deepseek.json (ANTHROPIC_AUTH_TOKEN field). Never store or
  commit it anywhere in this repo.
- Remove the temporary dump hooks when done. Commit nothing from this phase.

PHASE 1 - TTS BENCH (hard gate):
Bench candidates in this order: Pocket TTS (kyutai-labs/pocket-tts),
Supertonic-3 int8 via sherpa-onnx, Kokoro-82M via mlx-audio. Measure warm
time-to-first-audio on a ~15 word sentence and resident RAM. Pick the first
candidate that passes: warm first-audio < 1s and RAM < 1GB. Do NOT pause for
my input: pick by the numbers, save one sample WAV per candidate to bench/ so
I can ear-test later, and make the chosen engine a single swappable constant
in daemon.py so I can flip it without surgery. Stock voice, calm, slightly
fast. No cloning, no voice options, no Piper (macOS say is the stopgap if all
three fail).

PHASE 2 - DAEMON:
daemon.py, localhost-only HTTP server on port 7333. The daemon owns ALL logic:
- POST /speak {event, text, ts}, POST /stop, GET /health.
- Startup is bind-winner: if the port is already bound, exit quietly. Both CLIs
  may auto-start it simultaneously; first bind wins, loser's POST still lands.
- Text policy, async after accepting the request: strip markdown, drop code
  fences, never speak paths/URLs/code verbatim. Over ~350 chars: condense to
  ONE natural spoken sentence via DeepSeek, model deepseek-v4-flash,
  Anthropic-compatible endpoint https://api.deepseek.com/anthropic, token read
  at runtime from ~/.claude/settings.deepseek.json. API fails: speak the first
  sentence instead. Never block /speak's HTTP response on any of this.
- Loads the TTS model lazily on first /speak, exits after 10 min idle. First
  utterance after an idle exit may miss the 1s target: accepted, not a bug.
- Queue: one active utterance + one pending slot, newest pending wins. Every
  utterance gets a monotonic generation ID; /stop bumps the generation and a
  stale /speak result must never play after a newer /stop.
- Playback via afplay on a temp WAV; /stop kills the whole afplay process
  group, not just the parent.

PHASE 3 - HOOK SCRIPT:
hook.py, one script shared by both agents, a dumb forwarder and nothing more:
- Read stdin JSON. Stop event: take last_assistant_message, empty -> exit 0,
  else POST it raw to the daemon. UserPromptSubmit event: POST /stop.
- Auto-start the daemon if it is not running (spawn detached, don't wait).
- No cleanup, no condensing, no network beyond localhost. Stdlib only.
- It will be invoked as .venv/bin/python hook.py by absolute path (warm uv run
  measured 40-90ms on this machine, too close to the budget). Fire-and-forget,
  exit fast.

PHASE 4 - WIRE AND VERIFY:
Register Stop + UserPromptSubmit hooks for Claude Code in ~/.claude/settings.json
first, using the absolute .venv/bin/python command (no ~, no uv run), verify
end-to-end with a real turn, then Codex via [[hooks.Stop]] and
[[hooks.UserPromptSubmit]] in ~/.codex/config.toml, verify the same way.

INVARIANTS, never break these:
- Hooks must never block or slow the agent turn: exit within ~100ms, all
  cleanup, condensing, synthesis, and playback happens async in the daemon.
- Silence over noise: any failure (daemon down, API error, malformed payload)
  means no sound and exit 0. Never leak errors into the agent session.
- Never speak code, diffs, stack traces, or raw file paths.
- New user input interrupts current speech instantly; stale speech never
  resurrects after an interrupt.
- Everything runs local except the single DeepSeek condense call, and only the
  daemon ever touches the network or the token.
- Keep it small: no framework, no packaging, no config UI, no tests beyond the
  bench script. A handful of files. Personal tool, not a product.

One commit per phase, plain commit messages.
Mantra: verify the payload, bench the voice, then two small files and two config pastes. Everything else is ornament.
